# Skill: Prism 模块深度分析

## 触发条件

**以下情况触发：**

1. ✅ 写入知识库新模块文档时
2. ✅ 重构模块时需要更新文档
3. ✅ 用户要求"分析模块"、"写模块文档"、"详细分析"
4. ✅ 发现模块文档缺失或过时

**不触发的情况：**

- ❌ 仅修改 bug（使用 bug-to-wiki 技能）
- ❌ 仅修改配置（使用配置文档）
- ❌ 用户说"先不分析"

## 分析目标

对每个模块进行深度分析，输出文档必须包含：

1. **模块职责** — 核心功能、边界定义
2. **关键类和接口** — 类名、职责、关键方法、设计原理
3. **文件结构** — 头文件、源文件、每个文件的职责
4. **模块依赖** — 依赖的模块、被依赖的模块、依赖关系图
5. **文件依赖** — 哪个文件依赖哪个文件、头文件包含关系
6. **数据流** — 请求处理流程、关键调用链、数据转换
7. **设计原理** — 为什么这样设计、关键决策、性能优化
8. **关键代码片段** — 核心逻辑、重要配置项

## 分组策略

### 分组原则

1. **按调用链分组** — 相关模块放在一起分析
2. **每组 3-5 个模块** — 避免单组过大导致分析不深入
3. **核心模块单独分析** — agent、session、pipeline 等核心模块需要详细分析

### 默认分组方案

**组1：核心会话链路**
```
agent → session → worker → listener → balancer → dispatch → loader
```
分析重点：前端如何接收连接、如何分发到 worker、会话如何管理、配置如何加载

**组2：协议处理流水线**
```
protocol → recognition → pipeline → resolve
```
分析重点：协议如何识别、如何解析、如何路由、DNS 如何解析

**组3：传输与伪装层**
```
transport → stealth → crypto → multiplex → memory
```
分析重点：连接池如何管理、多路复用如何工作、伪装方案如何实现、加密如何工作、内存如何分配

### 自定义分组

用户可以指定：
- 分析哪些模块
- 分组方式
- 分析深度

## 分析流程

### 步骤 1：确认分析范围

```markdown
## 分析确认

1. 分析哪些模块？ {模块列表}
2. 分组方式？ {默认分组/自定义分组}
3. 分析深度？ {概要/详细/深度}
4. 输出路径？ {H:/{module}/detail.md}
```

### 步骤 2：读取源码

对每个模块：
1. 读取头文件目录 `include/prism/{module}/`
2. 读取源文件目录 `src/project/{module}/`
3. 读取相关测试文件 `tests/{module}*.cpp`
4. 读取相关配置文件

### 步骤 3：分析模块

对每个模块分析：

**1. 模块职责**
- 核心功能是什么
- 边界定义（做什么、不做什么）
- 在整体架构中的位置

**2. 关键类和接口**
- 类名、继承关系
- 关键方法签名和职责
- 接口设计原理

**3. 文件结构**
- 头文件列表和职责
- 源文件列表和职责
- 文件间依赖关系

**4. 模块依赖**
- 依赖的模块列表（输入）
- 被依赖的模块列表（输出）
- 依赖关系图（ASCII）

**5. 文件依赖**
- 头文件包含关系
- 源文件链接关系
- 编译依赖链

**6. 数据流**
- 请求处理流程
- 关键调用链
- 数据转换过程
- 状态变化

**7. 设计原理**
- 为什么这样设计
- 关键设计决策
- 性能优化点
- 权衡取舍

**8. 关键代码片段**
- 核心逻辑代码
- 重要配置项
- 关键算法

### 步骤 4：生成文档

按格式生成模块文档：

```yaml
---
title: {模块名} 模块详细设计
created: {YYYY-MM-DD}
updated: {YYYY-MM-DD}
type: module
tags: [{module}, architecture, design]
related: [{related modules}]
---
```

### 步骤 5：写入知识库

1. 创建目录 `H:/{module}/`（如果不存在）
2. 写入 `H:/{module}/detail.md`
3. 更新 `H:/wiki/index.md`
4. 更新 `H:/wiki/log.md`

### 步骤 6：验证

```markdown
## 验证清单

- [ ] 文件已创建在正确路径
- [ ] frontmatter 格式正确
- [ ] 包含所有必要部分（8 个部分）
- [ ] 至少 5 个 wikilinks
- [ ] 依赖关系图完整
- [ ] 数据流描述清晰
- [ ] 设计原理有解释
- [ ] index.md 已更新
- [ ] log.md 已更新
```
