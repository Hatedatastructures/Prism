# SOCKS5 协议完整文档

## 协议概述

SOCKS5 是一种网络代理协议，定义于 RFC 1928。它支持 TCP 和 UDP 代理，提供认证机制，支持多种地址类型

---

## 协议版本

- **版本号**: 0x05 (5)
- **默认端口**: 1080

## 1. 认证协商

### 请求格式

```txt
客户端 → 服务端:
+----+----------+----------+
|VER | NMETHODS | METHODS  |
+----+----------+----------+
| 1  |    1     | 1 to 255 |
+----+----------+----------+

字段说明:
┌──────────┬────────┬────────────────────────────────────────┐
│   字段   │  长度  │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   VER    │   1    │ 协议版本，固定为 0x05               │
│ NMETHODS │   1    │ 客户端支持的认证方法数量            │
│ METHODS  │ 1-255  │ 客户端支持的认证方法列表            │
└──────────┴────────┴────────────────────────────────────────┘

认证方法:
┌──────────┬────────┬────────────────────────────────────────┐
│   值    │  名称  │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   0x00   │ 无认证  │ 无需认证，直接进入请求阶段           │
│   0x01   │ GSSAPI  │ 通用安全服务认证 (RFC 1961)          │
│   0x02   │ 用户密码 │ 用户名/密码认证 (RFC 1929)           │
│   0x03   │ CHAP    │ Challenge-Handshake 认证              │
│   0x80-0xFE│ 私有  │ 私有方法，保留供自定义使用            │
│   0xFF   │ 无可用  │ 无可接受的认证方法                  │
└──────────┴────────┴────────────────────────────────────────┘

示例:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0x01 0x00                                               │
│ │    │    └── METHOD=0x00 (无认证)                            │
│ │    └── NMETHODS=1 (支持1种方法)                             │
│ └── VER=0x05 (SOCKS5)                                       │
└─────────────────────────────────────────────────────────────┘
```

### 响应格式

```txt
服务端 → 客户端:
+----+--------+
|VER | METHOD |
+----+--------+
| 1  |   1    |
+----+--------+

字段说明:
┌──────────┬────────┬────────────────────────────────────────┐
│   字段   │  长度  │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   VER    │   1    │ 协议版本，固定为 0x05               │
│ METHOD   │   1    │ 服务端选择的认证方法                │
└──────────┴────────┴────────────────────────────────────────┘

示例:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0x00                                                   │
│ │    └── METHOD=0x00 (选择无认证)                            │
│ └── VER=0x05 (SOCKS5)                                       │
└─────────────────────────────────────────────────────────────┘

拒绝示例:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0xFF                                                   │
│ │    └── METHOD=0xFF (无可用方法，拒绝连接)                  │
│ └── VER=0x05 (SOCKS5)                                       │
└─────────────────────────────────────────────────────────────┘
```

## 2. 请求格式

### 通用请求格式

```txt
客户端 → 服务端:
+----+-----+-------+------+----------+----------+
|VER | CMD |  RSV  | ATYP | DST.ADDR | DST.PORT |
+----+-----+-------+------+----------+----------+
| 1  |  1  | X'00' |  1   | Variable |    2     |
+----+-----+-------+------+----------+----------+

字段说明:
┌──────────┬────────┬────────────────────────────────────────┐
│   字段   │  长度  │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   VER    │   1    │ 协议版本，固定为 0x05               │
│   CMD    │   1    │ 命令类型                           │
│   RSV    │   1    │ 保留字段，固定为 0x00               │
│  ATYP    │   1    │ 地址类型                           │
│DST.ADDR  │ 变长   │ 目标地址，长度取决于 ATYP           │
│DST.PORT  │   2    │ 目标端口，大端序                   │
└──────────┴────────┴────────────────────────────────────────┘

命令类型 (CMD):
┌──────────┬────────┬────────────────────────────────────────┐
│   值    │  名称   │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   0x01   │ CONNECT │ 建立 TCP 连接到目标服务器            │
│   0x02   │  BIND   │ 绑定端口，接收反向连接              │
│   0x03   │ UDP_ASSOCIATE │ 建立 UDP 中继                 │
└──────────┴────────┴────────────────────────────────────────┘

地址类型 (ATYP):
┌──────────┬────────┬────────────────────────────────────────┐
│   值    │  名称   │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   0x01   │ IPv4   │ 4 字节 IPv4 地址                      │
│   0x03   │ DOMAIN │ 1 字节长度 + 域名 (最长255字节)         │
│   0x04   │ IPv6   │ 16 字节 IPv6 地址                     │
└──────────┴────────┴────────────────────────────────────────┘
```

### CONNECT 命令示例

```txt
示例: 连接 8.8.8.8:53 (DNS 服务器)
+----+-----+-------+------+----------+----------+
|VER | CMD |  RSV  | ATYP | DST.ADDR | DST.PORT |
+----+-----+-------+------+----------+----------+
| 05 |  01 |  00   |  01  | 08 08 08 08 | 00 35 |
+----+-----+-------+------+----------+----------+

解析:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0x01 0x00 0x01 0x08 0x08 0x08 0x08 0x00 0x35           │
│ │    │    │    │    └─────────────────┘ └─────────┘         │
│ │    │    │    │         DST.ADDR=8.8.8.8    DST.PORT=53    │
│ │    │    │    └── ATYP=0x01 (IPv4)                         │
│ │    │    └── RSV=0x00                                      │
│ │    └── CMD=0x01 (CONNECT)                                 │
│ └── VER=0x05                                                │
└─────────────────────────────────────────────────────────────┘

域名示例: 连接 example.com:443
+----+-----+-------+------+----------+----------+
|VER | CMD |  RSV  | ATYP | DST.ADDR | DST.PORT |
+----+-----+-------+------+----------+----------+
| 05 |  01 |  00   |  03  | 0B 65 78 61 6D 70 6C 65 2E 63 6F 6D | 01 BB |
+----+-----+-------+------+----------+----------+
                      │    └────────────────────────────────────┘
                      │         example.com (11字节)
                      └── 长度=0x0B (11)
```

### UDP_ASSOCIATE 命令示例

```txt
示例: 请求 UDP 中继
+----+-----+-------+------+----------+----------+
|VER | CMD |  RSV  | ATYP | DST.ADDR | DST.PORT |
+----+-----+-------+------+----------+----------+
| 05 |  03 |  00   |  01  | 00 00 00 00 | 00 00   |
+----+-----+-------+------+----------+----------+

解析:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0x03 0x00 0x01 0x00 0x00 0x00 0x00 0x00 0x00           │
│ │    │    │    │    └─────────────────┘ └─────────┘         │
│ │    │    │    │         DST.ADDR=0.0.0.0    DST.PORT=0     │
│ │    │    │    └── ATYP=0x01 (IPv4)                         │
│ │    │    └── RSV=0x00                                      │
│ │    └── CMD=0x03 (UDP_ASSOCIATE)                           │
│ └── VER=0x05                                                │

注意: DST.ADDR 和 DST.PORT 通常为 0，因为 UDP_ASSOCIATE

```

### BIND 命令示例

```txt
示例: 绑定端口等待反向连接
+----+-----+-------+------+----------+----------+
|VER | CMD |  RSV  | ATYP | DST.ADDR | DST.PORT |
+----+-----+-------+------+----------+----------+
| 05 |  02 |  00   |  01  | 00 00 00 00 | 00 50   |
+----+-----+-------+------+----------+----------+

解析:
┌─────────────────────────────────────────────────────────────┐
│ 0x05 0x02 0x00 0x01 0x00 0x00 0x00 0x00 0x00 0x50           │
│ │    │    │    │    └─────────────────┘ └─────────┘         │
│ │    │    │    │         DST.ADDR=0.0.0.0    DST.PORT=80    │
│ │    │    │    └── ATYP=0x01 (IPv4)                         │
│ │    │    └── RSV=0x00                                      │
│ │    └── CMD=0x02 (BIND)                                    │

BIND 流程:
1. 服务端绑定指定端口

```

## 4. UDP 数据报格式

### UDP 数据报结构

```txt
+----+------+------+----------+----------+----------+
|RSV | FRAG | ATYP | DST.ADDR | DST.PORT |   DATA   |
+----+------+------+----------+----------+----------+
| 2  |  1   |  1   | Variable |    2     | Variable |
+----+------+------+----------+----------+----------+

字段说明:
┌──────────┬────────┬────────────────────────────────────────┐
│   字段   │  长度  │                   说明                   │
├──────────┼────────┼────────────────────────────────────────┤
│   RSV    │   2    │ 保留字段，固定为 0x0000              │
│   FRAG   │   1    │ 分片序号，0x00 表示独立数据报         │
│  ATYP    │   1    │ 地址类型 (同请求格式)                │
│DST.ADDR  │ 变长   │ 目标地址                             │
│DST.PORT  │   2    │ 目标端口，大端序                     │
│   DATA   │ 变长   │ 用户数据                             │
└──────────┴────────┴────────────────────────────────────────┘
```

### UDP 数据报示例

```txt
示例: 发送 DNS 查询到 8.8.8.8:53
+----+------+------+----------+----------+--------------+
|RSV | FRAG | ATYP | DST.ADDR | DST.PORT |     DATA     |
+----+------+------+----------+----------+--------------+
|00 00|  00  |  01  |08 08 08 08| 00 35   | <DNS查询数据> |
+----+------+------+----------+----------+--------------+

解析:
┌─────────────────────────────────────────────────────────────┐
│ 0x00 0x00 0x00 0x01 0x08 0x08 0x08 0x08 0x00 0x35 <DATA>    │
│ │         │    │    └─────────────────┘ └─────────┘          │
│ │         │    │         DST.ADDR=8.8.8.8    DST.PORT=53     │
│ │         │    └── ATYP=0x01 (IPv4)                            │
│ │         └── FRAG=0x00 (不分片)                               │
│ └── RSV=0x0000                                              │
└─────────────────────────────────────────────────────────────┘

示例: 发送数据到 example.com:443
+----+------+------+----------+----------+--------------+
|RSV | FRAG | ATYP | DST.ADDR | DST.PORT |     DATA     |
+----+------+------+----------+----------+--------------+
|00 00|  00  |  03  |0B 65 78...| 01 BB   | <用户数据>   |
+----+------+------+----------+----------+--------------+
              │    └────────────────────────────┘
              │         example.com + 端口
              └── ATYP=0x03 (域名)
```

### UDP 分片说明

```txt
FRAG 字段:
┌──────────┬─────────────────────────────────────────────────────┐
│   值    │                       说明                           │
├──────────┼─────────────────────────────────────────────────────┤
│   0x00   │ 独立数据报，不分片 (最常用)                          │
│   0x01   │ 第一个分片                                         │
│   0x02-0x7F│ 后续分片                                         │
│   0x80   │ 最后一个分片                                       │
└──────────┴─────────────────────────────────────────────────────┘

注意: 大多数实现不支持分片，FRAG != 0x00 的数据报通常被丢弃
```

### TCP 控制通道的作用

```txt
┌─────────────────────────────────────────────────────────────────┐
│                 TCP 控制通道的作用                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 协商                                                        │
│     - 接收 UDP_ASSOCIATE 请求                                   │
│     - 返回 UDP Relay 地址和端口                                  │
│                                                                 │
│  2. 保活                                                        │
│     - 维持 TCP 连接                                             │
│     - 帮助 NAT 穿透 (保持 NAT 映射)                              │
│                                                                 │
│  3. 生命周期控制                                                 │
│     - TCP 断开时关闭 UDP Relay                                   │
│     - 防止资源泄漏                                               │
│                                                                 │
│  注意: TCP 连接不传输用户数据，仅用于控制                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### UDP 多路复用原理

```txt
┌─────────────────────────────────────────────────────────────────┐
│                 UDP 多路复用原理                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  一个 UDP Relay (一个 socket) 可以服务多个目标:                   │
│                                                                 │
│  客户端发送:                                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ UDP 数据报 1: 目标=8.8.8.8:53, DATA=DNS查询A            │   │
│  │ UDP 数据报 2: 目标=1.1.1.1:53, DATA=DNS查询B            │   │
│  │ UDP 数据报 3: 目标=192.168.1.1:123, DATA=NTP请求        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  UDP Relay 解析每个数据报的头部，根据目标地址转发:                 │
│                                                                 │
│  ┌─────────┐   目标A   ┌─────────┐                             │
│  │         │ ────────> │ 8.8.8.8 │                             │
│  │  UDP    │           │  :53    │                             │
│  │  Relay  │ <──────── │         │                             │
│  │         │   响应A   │         │                             │
│  │  (一个  │           ├─────────┤                             │
│  │  socket)│   目标B   │ 1.1.1.1 │                             │
│  │         │ ────────> │  :53    │                             │
│  │         │           │         │                             │
│  │         │ <──────── │         │                             │
│  │         │   响应B   │         │                             │
│  └─────────┘           └─────────┘                             │
│                                                                 │
│  关键: 每个数据报自带目标地址，Relay 根据地址路由                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 6. 地址格式详解

### IPv4 地址 (ATYP=0x01)

```txt
+------+----------+
| ATYP | ADDR     |
+------+----------+
|  01  | 4 字节   |
+------+----------+

示例: 192.168.1.100
┌─────────────────────────────────────────┐
│ 0x01 0xC0 0xA8 0x01 0x64                │
│ │    └────────────────────────────┘     │
│ │         192.168.1.100                 │
│ └── ATYP=0x01                           │
└─────────────────────────────────────────┘
```

### IPv6 地址 (ATYP=0x04)

```txt
+------+----------+
| ATYP | ADDR     |
+------+----------+
|  04  | 16 字节  |
+------+----------+

示例: 2001:db8::1
┌─────────────────────────────────────────────────────────────┐
│ 0x04 0x20 0x01 0x0D 0xB8 0x00 0x00 0x00 0x00                │
│ │    └─────────────────────────────────────────────┘        │
│ │         2001:0db8:0000:0000:0000:0000:0000:0001           │
│ └── ATYP=0x04                                               │
│                                                             │
│ 0x00 0x00 0x00 0x00 0x00 0x00 0x00 0x01                     │
│ └─────────────────────────────────────────────┘             │
│         (续上)                                              │
└─────────────────────────────────────────────────────────────┘
```

### 域名地址 (ATYP=0x03)

```txt
+------+----------+----------+
| ATYP | LEN      | DOMAIN   |
+------+----------+----------+
|  03  | 1 字节   | N 字节   |
+------+----------+----------+

示例: example.com
┌─────────────────────────────────────────────────────────────┐
│ 0x03 0x0B 0x65 0x78 0x61 0x6D 0x70 0x6C 0x65 0x2E 0x63 0x6F 0x6D │
│ │    │    └────────────────────────────────────────────────────┘ │
│ │    │         "example.com" (ASCII)                             │
│ │    └── LEN=0x0B (11 字节)                                      │
│ └── ATYP=0x03                                                   │
└───────────────────────────────────────────────────────────────┘

注意: 域名不以 NULL 结尾，长度由 LEN 字段指定
      域名最大长度 255 字节
```

## 7. 端口格式

```txt
端口始终使用大端序 (高字节在前)

示例: 端口 80
┌─────────────────────────────────────────┐
│ 0x00 0x50                               │
│ │    └── 低字节 = 0x50 (80)              │
│ └── 高字节 = 0x00                        │
└─────────────────────────────────────────┘

示例: 端口 443
┌─────────────────────────────────────────┐
│ 0x01 0xBB                               │
│ │    └── 低字节 = 0xBB (187)             │
│ └── 高字节 = 0x01 (1)                    │
│                                         │
│ 计算: (0x01 << 8) | 0xBB = 256 + 187 = 443 │
└─────────────────────────────────────────┘

示例: 端口 53
┌─────────────────────────────────────────┐
│ 0x00 0x35                               │
│ │    └── 低字节 = 0x35 (53)              │
│ └── 高字节 = 0x00                        │
└─────────────────────────────────────────┘
```

## 8. 实现注意事项

### 安全考虑

```txt
1. 认证
   - 生产环境应启用用户名/密码认证
   - 避免开放代理被滥用

2. 访问控制
   - 实现目标地址过滤
   - 限制可访问的端口范围
   - 防止访问内网地址

3. 资源限制
   - 限制 UDP 会话数量
   - 设置空闲超时
   - 限制数据报大小
```

### 性能考虑

```txt
1. UDP 中继
   - 使用单个 UDP socket 服务多个目标
   - 避免为每个目标创建新 socket

2. 内存管理
   - 预分配缓冲区
   - 避免频繁内存分配

3. 超时处理
   - UDP 会话空闲超时
   - DNS 解析超时
```

### 错误处理

```txt
1. 协议错误
   - 无效版本号
   - 不支持的命令
   - 无效地址格式

2. 网络错误
   - 连接失败
   - DNS 解析失败
   - 超时

3. 资源错误
   - 端口耗尽
   - 内存不足
```
